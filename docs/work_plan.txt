Установить VS Code и Claude Code
VS Code — программа для работы с кодом и файлами. Claude Code — дополнение, которое добавляет AI-помощника.

Что делать

1
Скачай VS Code с code.visualstudio.com
2
Установи как обычную программу
3
Установи дополнение Claude Code (подробно в курсе → Модуль 1)
4
Авторизуйся в Claude
Альтернатива: Можешь использовать Cursor — это похожая программа со встроенным AI. Принцип тот же.

Результат: программа открыта, Claude отвечает на сообщения.
Создать репозиторий на GitHub
GitHub — облачное хранилище для твоего проекта. Все файлы сохраняются, можно откатить изменения, и команда видит твой прогресс.

Что делать

1
Создай аккаунт на github.com (если нет)
2
Создай новый приватный репозиторий для своего проекта
3
Скачай его на компьютер и открой в VS Code
Как это сделать пошагово — в курсе, Модуль 2.

Результат: папка проекта открыта в VS Code.
Описание проекта
Этот раздел объясняет ЧТО ты делаешь. Claude прочитает его и поймёт твой проект.

Важно: Claude умеет создавать папки и файлы за тебя. Просто скажи что нужно — он сделает.

Что делать

1
Открой чат с Claude
2
Попроси создать структуру: «Создай папку docs/project/. В ней будем хранить описание проекта. Давай начнём с основного файла.»
3
Расскажи свою идею как другу — своими словами, без структуры
4
Попроси помочь оформить: что это, для кого, какую проблему решает, что будет уметь, чего НЕ будет
5
Спроси: «Что я мог упустить? Что обычно есть в таких проектах?»
6
Создавай дополнительные файлы для каждой большой темы
Результат: папка docs/project/ с файлами, описывающими твой проект.
Исследование аудитории
Кто будет этим пользоваться? Без понимания людей сделаешь продукт для себя, а не для них.

Если у тебя есть свои исследования

Сохрани их как .md файлы в папку research/
Claude увидит их и учтёт
Если делаешь проект из моих идей

Исследования уже скопированы в твой проект
Найди их в разделе «Исследования»
Что делать

1
Попроси Claude создать папку: «Создай папку docs/audience/. Будем исследовать целевую аудиторию.»
2
Расскажи что уже знаешь про своих пользователей
3
Попроси углубить: «Кто эти люди? Какие у них проблемы? Где они общаются?»
4
Для каждого типа пользователя создай отдельный файл с детальным описанием
Результат: папка docs/audience/ с файлами-портретами пользователей.

Анализ конкурентов
Что уже существует? Не чтобы копировать — чтобы понять своё место и отличия.

Что делать

1
Если есть свои заметки о конкурентах — положи в research/
2
Попроси Claude: «Создай папку docs/competitors/. Найди всё что существует похожего на мой проект.»
3
Для каждого конкурента создавай отдельный файл: что делают хорошо, что плохо, на кого рассчитано
4
Спроси: «Какие пробелы на рынке? Чем я могу отличиться?»
Результат: папка docs/competitors/ с файлами по каждому конкуренту.
